## 3. Findings & Remediation

### 3.1 Landmarks and Page Structure
- **Issue**: Missing explicit landmarks (`<header>`, `<main>`) and incorrect heading hierarchy.
- **Fix**: 
    - Updated `Layout.tsx` to use semantic HTML5 elements.
    - Standardized `h1` usage for page titles and maintained a logical descending order (`h2`, `h3`) for internal components.

### 3.2 Form & Input Accessibility
- **Issue**: Filter inputs (Search, Status Select, Switches) lacked accessible names.
- **Fix**: 
    - Added `label` props and `slotProps` to `TextField` and `Switch` components.
    - Ensured that the `Select` component correctly maps its label to the internal combobox via `SelectProps`.

### 3.3 Visual Contrast
- **Issue**: Primary brand colors and secondary text colors failed the 4.5:1 contrast ratio against the dark background.
- **Fix**: 
    - Brightened the primary accent color (`#6C63FF` → `#7C74FF`).
    - Increased luminance of secondary text colors (`#8B92A5` → `#A5ABB9`).
    - Verified all text now meets or exceeds WCAG AA thresholds.

### 3.4 Dynamic Content Announcements
- **Issue**: Screen reader users were not notified of background data updates or loading states.
- **Fix**: 
    - Implemented `aria-live="polite"` regions to announce search results count and loading status.
